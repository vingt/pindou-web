# 拼豆图纸生成器架构（Phase 0）

本文档定义首版模块边界，保证后续迭代可扩展且不互相侵入。

## palette

- 职责：主母库颜色模型、品牌映射模型、套装可用色集合。
- 关键约束：`MARD 221` 是唯一主母库；其余品牌只做映射层。
- 输入输出：输出 `MasterColor`、`BrandMapping`、`BrandPreset` 等结构给其它模块使用。

## pattern

- 职责：图纸网格、单元格颜色引用、生成模式相关类型定义。
- 关键约束：`PatternCell` 内部统一使用 `masterColorId` 存储。
- 输入输出：承接 palette 的主色体系，向 editor/export 输出标准图纸数据结构。

## editor

- 职责：编辑器 UI 与交互状态（工具、缩放、对照开关等）。
- 关键约束：不直接修改品牌映射规则；仅操作标准图纸结构。
- 输入输出：读写 `PatternProject`，并与 project 模块共享状态切片。

## export

- 职责：导出能力边界（图片/PDF/打印模板等，Phase 0 仅占位）。
- 关键约束：只消费标准化项目结构，不耦合生成算法细节。
- 输入输出：接收 `PatternProject` 与统计信息，输出不同导出格式（后续实现）。

## project

- 职责：项目级生命周期与状态总线（当前品牌、套装、模式、编辑器偏好）。
- 关键约束：作为跨模块协调层，不承载复杂业务算法。
- 输入输出：对 UI 暴露轻量 store，向各模块提供统一上下文。
