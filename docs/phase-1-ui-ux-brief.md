# 拼豆图纸生成器 · 阶段 1 界面清单（UX 骨架）

> 用途：定信息架构与每屏任务，**不画图**也可直接作为 Stitch / AI 生成界面的输入。  
> 产品形态：**Web 桌面优先**，中文 UI，数据主要保存在浏览器本机。

---

## 全局导航（所有工作区内页面共有）

- **顶栏左侧**：品牌标识（Logo + 产品名「拼豆图纸生成器」）→ 点击回首页 `/`
- **顶栏右侧**：主导航（四段）
  - **首页** → `/`
  - **编辑器** → `/editor`
  - **创作历史** → `/projects`
  - **设置** → `/settings`

**说明**：首页 `/` 为落地页，可有与内页略不同的顶栏，但四段导航语义一致。

---

## 一、`/` 首页（落地页）

### 用户要完成的 1～3 件事

1. **了解**产品是做什么的（照片 → 拼豆网格图纸）
2. **进入**编辑器开始干活
3. （可选）去创作历史找旧项目，或去设置改品牌数据

### 页面大块区域（上 → 下）

1. **顶栏**：Logo + 标题 + 四段导航
2. **Hero 区**：主标题、一句话说明、主按钮「进入编辑器」、次按钮「创作历史」
3. **能力介绍**：多列卡片（多品牌色板、生成与编辑、沉浸拼装、缺色与统计、导出、本地保存等）
4. **底部 CTA**：再次引导进入编辑器
5. **页脚**：版权/产品名 + 链到设置（或次要链接）

### 每块里最重要的一个入口

- Hero：**进入编辑器**（主按钮）
- 次：**创作历史**

---

## 二、`/editor` 编辑器

### 用户要完成的 1～3 件事

1. **上传**源图、设底板宽高、选品牌与色板套装、**生成**网格图纸
2. **编辑**格子（选工具、选颜色、改格内主色）、必要时**撤销/重做/重置**
3. 看**缺色/统计**、**导出**图纸；需要时进入**沉浸拼装**按色号专注施工

### 页面大块区域（左 → 右）

1. **顶栏**（工作区全局，同前）
2. **上方横条（仅编辑器）**：项目名称输入、**保存项目**
3. **左栏「参数」**（约固定宽）
   - 源图片：上传 / 更换
   - 底板尺寸：宽、高（珠数）
   - 拼豆品牌：多选其一
   - 色板数量 / 套装：多选其一 + **自定义色板**入口
   - 生成模式：严格套装 / 允许补色（及补色相关选项）
   - **高级生成设置**（可折叠）：处理模式、抖动、合并与清理等
   - 主操作：**生成图纸**、**比较方案**
3. **中栏「画布」**
   - 工具条：选择 / 画笔 / 橡皮 / 修复 / 替色 / 吸色器、撤销、重做、重置、缩放 + **沉浸拼装**
   - **沉浸拼装开启时**：额外一条「逐色」工具条（退出沉浸、上一色/下一色、显示全部、横向色号列表）
   - 主区域：可滚动的 **网格画布**（每格颜色 + 当前品牌色号文字）
4. **右栏**
   - **颜色面板**：当前选中色、图纸已用颜色列表（点选 = 画笔颜色）
   - **替色**：源色 / 目标色、执行替换
   - **缺色分析**（生成后）
   - **色号统计**（表格）
   - **导出**：PDF / PNG 等

### 每块里最重要的一个入口

- 左栏：**生成图纸**
- 中栏：**沉浸拼装**（有图之后）；日常核心是**网格**与**工具**
- 右栏：**导出**、**图纸已用颜色**（选色）

### 「沉浸拼装」模式（编辑器子状态，非独立路由）

- **目的**：画布区域尽量大，**每次只强调一种色号**的格子，其余变淡，方便实作
- **布局变化**：隐藏左参数栏 + 右栏 + 编辑器顶部的「项目名称/保存」横条；中栏画布占满主内容区
- **必须可见**：退出沉浸、切换色号、缩放（与逐色条）

## 三、`/projects` 创作历史

### 用户要完成的 1～3 件事

1. **浏览**已保存项目列表
2. **打开**某个项目回到编辑器继续改
3. **重命名 / 删除 / 导出**某个项目

### 页面大块区域

1. **顶栏**（全局）
2. **页头**：标题「创作历史」或等价文案、简短说明
3. **项目列表**：卡片栅格；每张卡片含：
   - 源图缩略图
   - 项目名称（可内联编辑）
   - 保存时间
   - 操作：**打开**（进编辑器）、**导出**、**删除** 等

### 每块里最重要的一个入口

- 单卡：**打开**（进入 `/editor` 并载入该项目）

---

## 四、`/settings` 设置

### 用户要完成的 1～3 件事

1. **维护**品牌列表（启用/排序/自定义品牌等，与实现一致）
2. **维护**某品牌的颜色数据（导入、编辑等）
3. **维护**某品牌的套装预设；必要时 **恢复默认设置**（清空本机品牌配置）

### 页面大块区域

1. **顶栏**（全局）
2. **页头**：标题「设置」、说明文字（数据存本机、恢复默认的含义）
3. **Tab**：**品牌** | **颜色** | **套装**
4. **Tab 内容区**：表格、表单、按钮（随 Tab 变化）
5. **全局危险操作**：**恢复默认设置**（放在页头或固定位置，需二次确认）

### 每块里最重要的一个入口

- 视用户任务而定：常见是 **恢复默认**（慎用）与各 Tab 内的**保存/应用**类按钮

## 五、关键用户流程 A：第一次从 0 生成图纸

1. 进入 **`/editor`**（或从首页点「进入编辑器」）
2. 在左栏 **上传源图片**
3. 确认或修改 **底板宽高**
4. 选择 **拼豆品牌**、**色板/套装**（若选自定义色板，需先选色）
5. 点击 **生成图纸**
6. 在中栏 **缩放/滚动**查看网格；在右栏查看 **缺色/统计**
7. 需要时点击 **导出**

**易错点**：未上传图就生成；严格套装 + 自定义色板但未选色；生成后想改参数会触发重新生成（需在文案或交互上可感知）。

---

## 六、关键用户流程 B：从创作历史打开再编辑

1. 进入 **`/projects`**
2. 在列表中找到项目，点击 **打开**
3. 进入 **`/editor`**，画布与项目数据已载入
4. 用 **画笔/橡皮**等修改，或 **保存项目** 更新历史记录
5. 需要时 **导出**

**易错点**：用户不知道「打开」会覆盖当前编辑器未保存的改动——若产品有提示策略，Stitch 可预留 **未保存离开确认** 的样式（可选）。

## 给 Google Stitch / AI 生成 UI 时建议附带的短提示（可复制）

```
中文界面，桌面 Web App，专业工具风格（清爽、留白足够、不要游戏化）。
主色 Indigo，中性灰 Slate，大圆角卡片 + 细边框，少用大阴影。
编辑器为三栏：左参数约 320px、中画布自适应、右面板约 340px。
沉浸拼装模式下隐藏左右栏，仅保留画布与顶部逐色工具条。
所有按钮文案保持中文，与上述路由与模块名一致。
```
